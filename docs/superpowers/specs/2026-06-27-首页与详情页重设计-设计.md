# 首页与详情页重设计 — 设计稿

- 日期：2026-06-27
- 范围：`web/`（信息流站首页 + 报告详情页），外加 research/stock 两个 skill 的板块标签约定（极小改动）
- 定位：**体验导向**的重设计——保留现在暖色衬线的阅读气质，外观只做必要精修，把力气全花在四个实测痛点上

## 背景与目标

现有站点（暖色纸底、衬线、留白、深色模式）外观已成熟，但用户点名四个体验痛点，且**四个全中**：

1. **手机端浏览/阅读** — 首页卡片偏大，一屏只看到 2–3 条；长报告全靠手指划。
2. **找内容难** — 只有类型筛选 + 全文搜索，没法按板块浏览；内容多了整页往下翻很累。
3. **长报告导航** — 详情页一长条到底，没有目录、章节跳转、阅读进度。
4. **信息密度太低** — 首页一屏能看到的条目太少。

目标：在不破坏现有阅读身份的前提下，让首页和详情页在这四件事上都明显变好。

## 关键设计决策（已与用户对齐）

| 决策点 | 结论 | 依据 |
|---|---|---|
| 卡片密度 | **平衡紧凑档**：类型徽标 + 标题一行 + 一句话结论一行 + 一行微元信息，一屏约 5–6 条 | 用户在 mockup 上选了推荐档；一句话结论保留（裁一行） |
| 板块数据来源 | **沿用 `related`，不新增 frontmatter 字段**；筛选与显示都取「与 5 大板块的交集」 | 26 篇研究里 24 篇的 `related` 已挂板块；`parse-note.js` 已把 `related`→`boards`；`feed.js` 已实现 `board:` 筛选 |
| 详情页目录/进度 | **走构建时注入**（`inject-report-nav.js` 那条路），运行时扫标题生成 | 所有已上线报告（含存量）立刻拥有，不用重跑生成；股票报告复用同模板也自动覆盖 |
| 整体外观 | 保留暖色衬线身份，不换皮 | 用户要的是解决体验，不是换风格 |

## 一、首页改动（`web/src` + `web/build`）

### 1.1 紧凑卡片
- 文件：`web/build/render-card.js`、`web/src/assets/feed.css`
- 新结构：`类型徽标(小描边)` + `标题(单行，超出省略)` + `一句话结论(单行 clamp)` + `微元信息(日期 · N 来源 · 板块)`。
- **板块显示对所有类型开放**（现在只有 `股票` 显示）。显示与筛选都用「板块交集」：只保留命中 5 大板块（光模块 / 机器人 / 算力 / AI应用 / 航天）的项，过滤掉 `related` 里的非板块双链（如「CPO 共封装光学」）。
- `data-type` / `data-boards` 继续保留供筛选（不变）。
- 卡片间用分隔线（行式），不再是大卡块。
- **3D 倾斜悬停**：高密度行式下逐行倾斜会发抖，改为更克制的悬停（左侧印章色竖条 + 轻微底色），更干净。手机端本就无倾斜。

### 1.2 板块筛选 + 两组筛选可叠加
- 文件：`web/src/index.template.html`、`web/src/assets/feed.css`、`web/src/assets/feed.js`
- 类型筛选行下面加一行**板块 chips**（5 个，`data-filter="board:光模块"`…）。`feed.js` 已支持 `board:` 分支。
- **筛选语义改造**：现在所有 chip 是同一互斥组。改成**两个独立组（类型 / 板块），用 AND 组合**——选「股票 + 算力」= 同时满足。需要重写 `feed.js` 的 `bindChips`：分别记录 active 类型与 active 板块，求交集决定卡片可见性。
- 把「筛选 + 计数 + 月分隔可见性」抽成**纯函数**（输入：entries + activeType + activeBoard → 输出：可见集合 / 各月计数），DOM 层只做薄绑定，便于单测（现在 `feed.js` 无单测）。

### 1.3 条目计数 + 按月分组
- 文件：`web/build/render-index.js`、`web/src/index.template.html`、`feed.css`、`feed.js`
- 顶部显示「共 N 篇 · 按时间倒序」。
- 信息流按月插入分隔行 `2026 · 六月`（卡片已按新→旧排序，跨月边界插入）。
- 筛选时：重算可见卡片 → **隐藏没有可见卡片的月分隔** → 更新计数（显示当前可见数）。
- 搜索态（输入框有词、feed 整体隐藏、改用 results 容器）下分隔/计数不参与。

### 1.4 顶栏紧凑 + 吸顶
- 文件：`web/src/index.template.html`、`feed.css`
- brand 收成一行（印章 + 「searchX · 调研档案」+ 右侧「+ 提交」）。
- 搜索框 + 类型/板块筛选 + 计数包进 `position:sticky; top:0` 容器，滚动时筛选始终够得着；滚动后加细底边分隔。手机、电脑都适用。
- 保留提交弹窗入口与全部现有行为。

### 1.5 浮动按钮：保留 + 宽屏贴列
- 文件：`web/src/assets/feed.css`
- **保留现有两个按钮**：刷新（常驻）+ 回到顶部（滚动后浮现）。仅修定位，不删不换。
- **宽屏贴列定位**：现在按钮死贴视口右缘（`right:20px`），宽屏上正文是居中窄列、两侧大留白，按钮被甩到离正文很远的屏幕最右角。改为跟随正文列：
  `right: max(20px, calc((100vw - var(--measure)) / 2 - 56px))`。
  窄屏（手机）维持离右缘 20px；视口变宽后按钮移入右侧留白、紧挨正文列右缘（约 12px 间距，`56px ≈ 按钮 44 + 间距 12`）。
- 需给首页 `:root` 补 `--measure: 46rem`（首页 `.wrap` 当前 max-width 即 46rem）。

## 二、详情页改动（`web/build/inject-report-nav.js`）

全部在构建时注入到 `web/dist/r/<dir>/index.html`；原始 `research/<dir>/report.html`（归档 / Obsidian 用）保持纯净不动。注入的是真实站点 CSS/JS，可用 `position:fixed`。

### 2.1 顶部阅读进度条
- 顶部一条细线（印章色），随滚动按文档高度填充。
- reduced-motion 下去掉过渡动画，仍正常填充。

### 2.2 自动目录
- **运行时扫描**这些锚点并按文档顺序建目录：`.tldr`(核心结论)、`.findings`(关键发现)、`main h2`(正文各章)、`section.risks`(风险与争议)、`.glossary`(名词小抄)、`section.sources`(来源清单)。缺 `id` 的补 `id`。
- **电脑端（约 ≥1100px）**：正文左侧那片大留白里放 `position:fixed` 吸顶侧栏目录，当前章节高亮（scroll-spy），点击平滑跳转；窄于阈值自动隐藏，退回浮层。
- **手机/窄屏**：右下角加「目录」浮动按钮（与现有 回首页 / 回顶 并排，相应上移：回顶 74px、目录 128px 之类），点开底部浮层列出各节，点哪节跳哪节并关闭，浮层底部显示阅读进度 %。
- 浮层可访问性复用首页弹窗手法：Esc 关、点遮罩关、焦点处理、`aria` 属性。
- 不破坏现有注入（回首页按钮、回顶、viewport 锁定、表格/图片自适应 CSS）。

### 2.3 浮动按钮：保留现有 + 新增目录 + 宽屏贴列
- 文件：`web/build/inject-report-nav.js`
- **保留现有两个按钮**：返回主页 + 回到顶部。本次只在这一摞上**新增「目录」按钮**（窄屏才显示；电脑端有侧栏目录时隐藏）。
- 三个按钮从下到上：回主页（bottom 20px）、回顶（74px）、目录（128px）。
- **宽屏贴列定位**（与首页 1.5 同规则）：`right: max(20px, calc((100vw - var(--measure)) / 2 - 56px))`，`--measure` 报告页已为 42rem。窄屏离右缘 20px，宽屏紧挨正文列右缘。
- **左侧目录侧栏对称**：用对称算式贴正文列左缘——`left: max(20px, calc((100vw - var(--measure)) / 2 - <侧栏宽 + 间距>))`，与右侧按钮左右对称、都挨着正文。

## 三、板块数据（research + stock skill，极小改动）

- 现状：26 篇研究 24 篇已在 `related` 挂了板块，板块筛选近乎现成。
- 改动：在 `.claude/skills/research/SKILL.md` 与 `stock/SKILL.md` 里**明确写**——主题确与 5 大板块之一相关时，把板块名放进 `related`（本就是惯例，写成约定）。**不新增字段**。
- 轻量回填：审一遍未命中的 2 篇及个别可能漏挂的，确有关联就补进 `related`（大多已 OK）。
- 5 大板块作为**单一常量**（建议 `web/build/boards.js`），被 `render-card.js`（显示交集）与板块 chips（模板）共用，避免清单漂移。

## 四、不做（YAGNI）

- 不新增 frontmatter 字段（沿用 `related`→`boards` + 交集）。
- 不做正文内引用脚注 ↔ 来源清单的双向跳转（超范围）。
- 归档 / Obsidian 的 `report.html` 不动（目录 / 进度仅注入 dist 副本）。
- 不做整体视觉换皮（保留暖色衬线身份）。
- 不做无限滚动 / 分页（按月分组 + 计数已够；内容规模还小）。

## 五、测试

- 新建纯函数测试：两组筛选 AND、各月计数、月分隔可见性、搜索态边界（把逻辑从 `feed.js` 抽出）。

- `inject-report-nav.test.js`：注入了进度条 / 目录骨架 / 目录按钮且带可访问性属性；现有断言（回首页 / 回顶 / viewport / 表格 CSS）不破。目录内容运行时生成，单测只验注入骨架与属性。
- `build.test.js`：端到端仍绿。

## 六、风险与边界

- 筛选 × 月分隔联动：空月分隔必须隐藏；搜索态下分隔 / 计数不参与。
- 桌面侧栏目录与窄屏阈值：避免与居中正文列重叠；低于阈值退回浮层。
- 手机目录浮层：焦点陷阱 / Esc / 点遮罩关闭 / `aria`，复用首页弹窗实现。
- 深色模式：所有新元素一律用站点 / 报告既有 CSS 变量，不写死颜色。
- reduced-motion：进度条过渡、平滑滚动、卡片动画全部降级。
- `related` 混入非板块双链：显示与筛选都取 5 大板块交集，脏标签不外露。
