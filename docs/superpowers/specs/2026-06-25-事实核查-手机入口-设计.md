# 事实核查 — 手机入口（分阶段）设计稿

> 2026-06-25。`/factcheck` skill 已做好（见 `2026-06-25-事实核查-skill-设计.md`），但它的入口在 Claude Code 对话框——人在外面拿手机用不了，等于废了一半。本稿补一个**手机能用的私密入口**：手机录入 → 后台 `claude -p` 自动跑 → 结果同步回手机 Obsidian。
> 一句话：**照 `runner` 的样子，在现有 Cloudflare + 常驻机基础上，复制一条产出私密、不上线的核查流水线。**

## 背景 / 动机

核查是高频、且常在外面（手机）发生的需求。纯 skill 入口在对话框、手机不可用。需要：手机录入一句话 / 链接（阶段 2 加截图）→ 后台自动核查 → 结果出现在手机 Obsidian。

项目已有一条几乎同构的成熟链路——`runner`（常驻 Mac mini，`claude -p` headless 跑 `/research` + 自动产出，配齐单实例锁 / 幂等 / 失败重跑）。本方案**复用这套架构**，做一条姊妹流水线。

## 已拍板决策

1. **复用 runner 架构**：`claude -p` headless 跑 skill、常驻机定时轮询。已验证可行（在跑 /research）。
2. **入口 = 独立私密核查页**：不混进现有给朋友用的公开提交页；凭**作者一人的密钥**进入（类似 `ADMIN_KEY` 纯密钥，**不用**朋友那套 token 白名单——只有你用，从简）。
3. **任务通道 = Cloudflare KV**（已有 `INTAKE_KV`），**绝不进公开 GitHub Issue**（私密内容）；阶段 2 的截图存 **R2**。
4. **独立核查 runner**：复用现有锁 / `claude -p` spawn / 邮件底层，但主流程独立（取 KV 而非 Issue、不上线、不查重）。
5. **产出回手机的命门 = Obsidian Sync**（已确认采用）：Mac mini 装 Obsidian、登录同库、开 Sync；runner 写本地库 → Sync 推云 → 手机。注意 Mac mini 上 Obsidian 客户端需**保持开启**（App 内同步，非系统级文件同步）。
6. **分阶段**：阶段 1 先做 文本 + 链接 闭环（快速可用，覆盖政策 / 公众号链接 / 新闻 / 话题）；阶段 2 补截图（R2 + 上传 + 喂图，最复杂的一块）。

## 总体流程（阶段 1）

```
手机浏览器 → 私密核查页（文本 + 链接，凭密钥进入）
   │  POST {key, text, link}
   ▼
Cloudflare Worker（在 intake-worker 加私密路由）
   ├─ 密钥校验（safeEqual + 失败限流，复用 admin 那套）
   ├─ 校验 / 清洗内容
   └─ 任务写入 KV：check:<id> = {text, link, status:"pending", createdAt}
   ▼
核查 runner（Mac mini 常驻，独立流程）轮询
   ├─ 取 status=pending 的任务
   ├─ claude -p "/factcheck <text 或 link>"
   ├─ factcheck 产出到 <OBSIDIAN_VAULT>/Factcheck/（不上线、不进仓库）
   ├─ KV 标 status=done
   └─（可选）发"核查完成"邮件提醒
   ▼
Obsidian Sync → 手机 Obsidian 看到核查笔记
```

## 阶段 1 组件详解

### 1. 私密核查页（`web/`）
- 新增静态页（`check.template.html` + `assets/check.js`），**noindex、站内不放入口链接**。
- 纯密钥进入：输对密钥前不渲染表单（同 admin 页套路）；密钥存 sessionStorage、走请求头不进 URL。
- 字段：核查内容（文本，必填）+ 链接（可选）。提交 POST 到 Worker，带密钥头。
- 走现有 CI 自动部署——它是公开静态页，但**没密钥进不去、且不被索引**，真正的锁在 Worker 端。

### 2. Worker 私密路由（`services/intake-worker/`）
- `POST /check`：密钥校验（复用 `safe-equal.js` + 失败限流）→ 校验 / 清洗 → 写 KV `check:<id>`。
- `GET /check/pending`（凭 runner 共享密钥，类似现有 `/sub`）→ runner 取待处理任务。
- `POST /check/<id>/done`（凭 runner 密钥）→ 标完成。
- KV 键 `check:<id>` 带 TTL（如 7 天，防堆积）。

### 3. 核查 runner（`services/`，新目录 `check-runner/`）
- 复用：单实例锁、`Bun.spawn claude -p`、nodemailer、config 加载。
- 主流程：轮询 `/check/pending` → 每条 `claude -p "/factcheck …"` → 据 claude 退出码 / 新增笔记判成功 → `POST /check/<id>/done` →（可选）发信。
- 独立 launchd 定时（仿 `com.searchx.runner.plist`）。
- **不查重、不上线**（factcheck skill 本身就不 push）。

### 4. factcheck skill 微调
- 接"无人值守"上下文：runner 跑时无人在场——skill **不反问**、产出后**不必在对话里给摘要**（对话没人看），直接落 Obsidian 笔记即可。加一句条件说明即可。

### 5. Mac mini 部署
- 装 Obsidian、登录同一 vault、开 Sync、保持 App 开启。
- 确认 Mac mini 的 `CLAUDE.local.md` 里 `OBSIDIAN_VAULT` 指向该 vault。
- 装核查 runner 的 launchd 定时。
- Worker 端配两把密钥（核查页密钥 + runner 共享密钥）。

## 阶段 2：截图（概要，本期不做）
- Worker `POST /check` 支持图片 → 存 R2；KV 任务记 R2 key。
- runner 取任务时从 R2 下载图片到本地临时文件。
- `claude -p` prompt 引用该本地路径，factcheck skill Step 0 接"图片路径 + 文本"混合输入。
- 前端核查页加图片上传控件。

## 隐私 / 安全
- **私密内容绝不进公开 Issue / 公开仓库**：任务在 KV、产出在 Obsidian（仓库外）。
- **密钥鉴权**：核查页用作者一人密钥（safeEqual 定长比较、失败按 IP 限流锁定、走请求头不进 URL、存 sessionStorage）。runner 取任务用独立共享密钥（同 `/sub` 套路）。
- **凭据永不入库**：新密钥走 Worker secret + 本机 `.env`。
- runner 的 `claude -p` 仍 `bypassPermissions` headless——但只处理你自己经密钥提交的内容，且 factcheck 不 push、不写仓库。
