# 股票深度分析 skill — 设计（2026-06-04）

把用户的《股票研究提示词》（未来 13 周走势情景 + 条件式触发，BLUF 结论先行，A–M 结构）做成一个 skill，融入 searchX 现有深度调研流水线：自动识别"单只股票"请求并路由过去，**纯 WebSearch 研究**（不接本地数据库），**检索结果先核验、再出观点**，产出三件套并自动上线、落 Obsidian，公开站作为新类型「股票」展示。

参考输入：用户提供的提示词原文（角色=投研逻辑审查员+信息核验员；A 一屏结论 → B 公司快照 → C 财务健康 → D 护城河 → E 13周事件日历 → F 政策宏观 → G 产业链 → H AI关联 → I 预期差 → J 8条逻辑链 → K 三情景 → L 条件式风控 → M 逻辑自查）。原文已存 Obsidian `饺子的旷野/AI相关/提示词/股票研究提示词.md`。

## 核心原则（本轮重新摆正）

**深度来自 WebSearch 打一手来源 + A–M 方法论 + 推理，不来自本地数据库。** 本地 Stocks 库的新闻/财务对研究无增量价值（一手披露经 web 取更新、更全、更可溯源）。v1 **不接任何本地数据/Tushare/bridge**。"从库里取几个精确量化数字"作为可选增强，**推迟**（见 §8）。

## 1. 形态与路由

- 新建 `.claude/skills/stock/SKILL.md`：装 A–M 框架 + 股票专属来源规则 + **核验闸** + 单只护栏。基于用户提示词原文改写适配。
- `/research` 的 **Step 0** 增加识别规则：当 `$ARGUMENTS` 指向**单一上市公司股票**（含代码、明确个股名、或"怎么看 XX 这只票 / XX 股价走势"等信号）→ 委派给 stock skill 执行，**跳过通用 Step 3**。
- **复用** research 的 Step 1（先校正前提）、Step 2（检索纪律与来源优先级）、Step 4/5/6（三件套产出 + Obsidian 双链 + 自动同步上线）。stock skill 的内部流程 = **检索 → 核验（§3）→ A–M 分析与结论 → 三件套输出**。
- **单只股票护栏**：一次只分析一只。请求含多只 → 无人值守取主标的、显式标注其余超出范围；交互式运行可反问澄清。

## 2. 研究方法 = A–M（纯 WebSearch）

skill 主体即用户提示词的 A–M 框架，落实其全部硬规则：

- **来源优先级**（强约束）：一手披露（交易所公告/定期报告/问询函/SEC filing）＞ 公司官方（IR/路演/电话会纪要/互动易/上证e互动/深交所互动）＞ 权威媒体（仅确认事件时间点）＞ 财经媒体（仅线索）；**不采用**自媒体/雪球股吧个人观点/未署名小作文与研报结论。
- **检索纪律**（继承并特化 research Step 2）：拆成子问题逐条 WebSearch；一手来源优先；时间一律换算北京时间。
- **结论先行（BLUF）**：报告开头一屏结论（A 节），每章节内部先小结再展开。
- **强标注**：关键事实标"来源类型 + 发布日期（北京时间）"；每个结论标置信度（高/中/低，理由 ≤2 句）；明确"财报期末口径 vs 当前经营"的滞后风险。
- **远期颗粒度**：9–13 周事件可降颗粒度但保留关键日期锚点。

## 3. 核验闸（出观点前的硬步骤 · 严谨性核心）

**检索完、写任何结论之前，对拟采用的每个关键事实过一遍核验；过不了的不许进结论。** 这是独立于网站代码测试（§7）的、针对**研究内容本身**的关卡——也是项目严谨性的核心。

逐项核验：
1. **可溯源**：每个数字/日期/事件/引述都能落到**具名一手来源**；回到一手源本身复核，不止步于聚合器/二手转载。
2. **交叉验证**：重要论断尽量 **≥2 个独立来源互证**；仅单一来源 → 标低置信度或降级为线索，不当定论。
3. **冲突处理**：来源冲突按优先级裁决并**显式标注冲突**，不替读者武断下结论。
4. **抓典型错误**：数据滞后（财报期末口径 vs 当前经营/订单/价格）、张冠李戴（概念关联 vs 业务关联）、把题材当基本面、单一来源传闻/小作文。
5. **数字与日期复算**：关键财务/价格/事件日期二次核对；查不到就写"信息缺口 + 应去查的位置（公告标题/报告名/关键词）"，**绝不编造**。
6. **独立复核最关键项**：对支撑 A 节方向判断的 1–2 个核心驱动/风险，**独立再检索/再核一遍**（可另起子代理做对抗性复核）后再定稿。

闸门规则：**任一关键事实过不了核验 → 降级为"信息缺口"或下调置信度，绝不进 A 节结论。** 先核验事实（本步），后写结论，最后 M 节自查逻辑——**本步管"检索到的对不对"，M 节管"推理有没有漏洞"，两道关分工不重叠。**

## 8. 部署

- v1 纯 web，**不依赖 Stocks 库/Mac mini**，跟 `/research` 一样在当前 MBP 的 runner 上即可跑、即可出真实报告。
- **推迟项（v2 可选，非本轮）**：若日后真觉得报告缺少精确量化（最新价/振幅、估值 PE/PB/PS+历史分位、资金流向北向/融资融券），再加一个**极薄、可选、缺则跳过**的量化补充，复用 Stocks 项目数据，那时面向 Mac mini 就地取数。Stocks 数据能力已探明并记在记忆 [[stock-analysis-skill]]，备查。

## 9. 非目标（YAGNI）

- v1 不接本地数据库 / Tushare / bridge / venv / 机器开关。
- 不取本地新闻表（web search 更强）。
- 不强行调 Tushare 财务三表（财务从一手 filing 经 web 取，更可核）。
- 不做语义检索（本轮已搁置）。
- 不支持多只股票对比成篇（单只护栏）。
- 表单不加结构化股票字段（只加提示文案）。
